EXPLANATION

The fundamental problem of color palette generation is trying to approximate a continuous space (RGB) with discrete points. Most color palette generators either sample randomly from images or rely on standard color schemes (monochromatic, complementary, etc.), often combining them to create a limited set of colors (like 16; choose 1 -> calculate 3 -> 4; take 1 of 4 -> calculate 3 -> repeat for remaining 3 -> 16). The results are fine, but I wanted to find a way to do this that wasn't just arbitrary selection or simple interpolation between points.

The core idea is treating the RGB cube as a metric space. Each color is just a point (R,G,B), so we can choose to define the distance between colors as the maximum difference in any component: max(|R‚ÇÅ-R‚ÇÇ|, |G‚ÇÅ-G‚ÇÇ|, |B‚ÇÅ-B‚ÇÇ|). This is the Chebyshev distance, and it creates these interesting octahedral spheres of influence around each color.
Why did I choose it over the Euclidean distance? If you want to optimize the color space covered by a palette of arbitrary size, you want to focus on the largest difference in each dimension separately. This means two colors are considered "far apart" if any one channel has a large difference, regardless of the other channels, so a palette created with consideration of the Chebyshev distance ensures that even if two colors are similar in R and G but very different in B, they are considered distinct.
Using this distance metric, we can frame palette generation as a coverage problem: given two anchor points (your chosen colors), how do we place 2‚Åø points (specifically, the 2‚Åø‚àí2 remaining after anchoring) to minimize the maximum distance any possible color could be from its nearest palette color? This reminds me ‚Äî I should probably mention why I chose to exclusively focus on palettes of size 2‚Åø. I've been fixated on computer graphics lately, so while I was reading about vintage computer hardware graphics and subjecting myself to torturous manual labor (creating a graphics API in C), I decided on constraining the space with a binary partition. Why? Binary chunks are chad.
